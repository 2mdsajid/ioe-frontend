# ⚙️ IOE Locus

### The Precision Diagnostic Terminal for Engineering Candidates

**IOE Locus** is a high-performance entrance preparation platform engineered specifically for **TU Pulchowk**, WRC, ERC, and Thapathali engineering aspirants. Developed under the **Edulocus Initiative**, the platform replaces traditional rote learning with technical mastery through data-driven diagnostics and CBT-standard simulations.

---

## 🚀 Technical Features

* **System Diagnostics**: Real-time feedback on mock modules with precise accuracy and load-speed tracking.
* **Module Specialization**: Deep-dive analytics across Mathematics (50 Marks), Physics (45 Marks), Chemistry (25 Marks), and English (20 Marks).
* **High-Yield Question Bank**: Over a decade of IOE past questions categorized by subject weightage and chapter logic.
* **Secure Candidate Terminal**: Industry-standard data encryption for performance logs and candidate identity.
* **Pro Membership**: Exclusive access to advanced full-length simulators and automated weakest-node detection.

---

## 🛠️ Tech Stack

Optimized for high-concurrency and low-latency diagnostic performance:

* **Framework**: [Next.js](https://nextjs.org/) (App Router)
* **Language**: [TypeScript](https://www.typescriptlang.org/) for type-safe engineering logic
* **Styling**: [Tailwind CSS](https://tailwindcss.com/) with a professional "Slate & Indigo" light theme
* **Components**: [Shadcn/UI](https://ui.shadcn.com/) for precision-engineered interfaces
* **Authentication**: [Lucia Auth](https://lucia-auth.com/) with Google OAuth integration
* **Database/ORM**: [Prisma](https://www.prisma.io/) with Zod for robust schema validation

## 📂 Project Structure

```bash
├── app/                  # Next.js App Router (Diagnostic Dashboard, Exams)
│   ├── dashboard/        # Candidate analytics and performance logs
│   ├── premium/          # Pro tier enrollment and system access
│   └── (auth)/           # Secure terminal access and authentication
├── components/           # Technical UI components (NodeGrid, FormulaBank)
├── lib/                  
│   ├── actions/          # Server actions for diagnostics and feedback
│   ├── schema/           # Zod definitions for marking logic and subjects
│   └── data/             # Static technical content and IOE metadata
└── public/               # Brand assets and system documentation

```
